تعريف المشروع (موجز وواضح)

اسم المشروع: SERA — منصة ذكية مدمجة للتطوير الشخصي والتقييم البرمجي.
جوهر المشروع: نظام ذكي يجمع بين التحليل النفسي وتقييم المهارات التقنية لتوليد مسارات تعلم شخصية جداً، متكاملة مع دعم نفسي يومي، اختبارات برمجية، ومتابعة تقدم دقيقة. (المصدر: المستند المرفق). 

Plan

أهداف المشروع (محددة وقابلة للقياس)

فهم حالة المستخدم النفسية والمعرفية في أول جلسة وتوليد «خريطة ذهنية» و«بصمة مهارية».

تقديم مسار تعلم برمجي مخصص يزيد قابلية إتمام المسارات بنسبة ≥ 40% خلال 90 يوم.

دمج دعم نفسي مصغر (check-ins، تمارين قصيرة) يقلل مستويات الإجهاد لدى المستخدمين النشطين بمعدل ملموس.

قياس وتحسين قدرة حل المشكلات لدى المستخدم عبر اختبارات دورية مع تقارير تقدم. 

Plan

نطاق العمل (ما سيتم تسليمه فعلياً)

مواصفات الوظائف الأساسية: تسجيل/onboarding مزدوج (نفسـي + تقني)، لوحة تحكم يومية، محرك دورات مخصص، نظام تقييم برمجي متنوع، نظام دعم نفسي يومي، مميزات اشتراك مرقمة (PRO, PRO MAX, MASTER, GOD-TIER). 

Plan

وثيقة UX شاملة مع تدفقات المستخدم، شاشات رئيسية، حالات الحواف، متطلبات محتوى.

مصفوفة قبول (acceptance criteria) واختبارات وظيفية لكل ميزة.

مواد التسليم للمطور: وصف وظيفي مفصل، خرائط تجربة المستخدم، سيناريوهات الاستخدام، قائمة بيانات أولية ومخرجات لكل شاشة. (لا تتضمن تعليمات تقنية أو أدوات تنفيذيّة). 

Plan

المستخدم المستهدف / Personas

Primary Persona — محمود / عمر

سن: مراهق/شاب (16–25)

مشكلة أساسية: تشتت، قلق من المستقبل، يريد دخول البرمجة لكنه غير منظم.

حاجة: خطة واضحة، متدرجة، ودعم نفسي بسيط يبقيه مستمرًا. 

Plan

المتطلبات الوظيفية (بترتيب الأولوية)

Onboarding (3 خطوات):

بصمة نفسية (أسئلة تحليلية، قراءة نمط الكتابة).

بصمة مهارية (اختبار تقني سريع بمزيج أسئلة متعددة الاختيارات ومقاطع كود بسيطة).

بصمة مستقبلية (أسئلة طموح/أهداف).

الناتج: Mindprint Map, Coding Genome, Life Trajectory. 

Plan

Dashboard موحّد:

عرض يومي يجمع مهام نفسية وتقنية.

مؤشرات طاقة/تركيز يومية، مقترحات سريعة “What should I learn today?”. 

Plan

محرك الدورات المخصّص:

يولد مسارات تعليمية يومية/أسبوعية مبنية على بيانات المستخدم النفسية والتقنية.

محتوى: نص، فيديوهات قصيرة، اختبارات صغيرة، مشاريع مصغرة. 

Plan

نظام التقييم (Coding Evaluation):

اختبارات تصحيحية: Debugging, Algorithms, Problem Solving.

توليد خريطة نقاط الضعف Heatmap وتقارير قابلة للمطالعة. 

Plan

تجربة يومية ذكية (Daily Journey):

Morning Scan → Daily Lesson → Mini Test → Short Psychological Task → Evening Reflection. 

Plan

ملاحظات فورية وتحفيز (Micro-feedback):

بعد كل مهمة فنية/نفسية، رسالة قصيرة تشرح النتيجة والنصيحة التالية. 

Plan

خطط اشتراك مميزة (وظائف وصفية فقط):

طبقات PRO وPRO MAX وMASTER وGOD-TIER مع خدمات مميزة (تحليلات أعمق، مدرب افتراضي، توليد كورسات مخصص). 

Plan

تجربة المستخدم (UX) — تدفق مفصّل وشاشات رئيسية

(التسلسل أدناه يجب أن يكون وثيقة مرجعية واضحة للمهندس لفهم ماذا يجب أن يعمل التطبيق دون ذكر كيف يُنفّذ)

فتح التطبيق / شاشة ترحيب

رسالة تمهيدية قصيرة توضح الهدف. زر «ابدأ». 

Plan

Onboarding — ثلاث شاشات متعاقبة

شاشة 1: أسئلة نفسية قصيرة. مخرجات: Mindprint.

شاشة 2: أسئلة تقنية قصيرة (متنوع: اختيار/تصحيح كود). مخرجات: Coding Genome.

شاشة 3: أسئلة رؤية مستقبلية وتحديد أهداف. مخرجات: Life Trajectory.

نهاية: بطاقة «ملخص أولي» تعرض نتائج موجزة ونصيحة فورية. 

Plan

Dashboard — Intelligence Hub

أجزاء: ملخص الطاقة اليومي، مهام اليوم (نفسية وتقنية)، مؤشر التقدم، زر «ما الذي يجب أن أتعلمه اليوم؟»، بطاقة التقدم الأسبوعي. 

Plan

شاشة الدرس / المهمة

عرض درس مختصر، تمرين عملي صغير، اختبار سريع (3 أسئلة). زر «أكمل» مع تلخيص النتيجة فوراً. 

Plan

شاشة التقييم (Assessment Detail)

نتائج مفصلة: مهارات، خريطة ضعف، توصيات فورية لمسارات تصحيحية. 

Plan

شاشة الدعم النفسي المصغّر

تمارين تنفّس/مذكرات قصيرة/تذكير بأخذ استراحة. متوفّرة أثناء حل الاختبارات إذا رُصدت علامات توتر. 

Plan

شاشة التقارير / تراكم الأداء

تقارير أسبوعية وشهرية: تطور التركيز، مستوى الإحباط، تقدم الكفاءات التقنية. 

Plan

شاشة الإعدادات/الملف الشخصي

عرض ملخص Mindprint وCoding Genome وLife Trajectory، إمكانية تحديث الهدف. 

Plan

سيناريوهات استخدام (User Journeys) — أمثلة عملية

محمود، يوم عادي: يفتح التطبيق → Morning Scan يقول إن طاقته منخفضة → يقترح درس قصير + تمرين نفسي 2 دقيقة → بعد التمرين يحل تمرين ويشعر بتحسن → Evening Reflection يعرض تقدم يومي. 

Plan

مستخدم في اختبار ضغط: أثناء حل اختبار تطول مدة الإجابة وتزداد الأخطاء → النظام يعرض نافذة اقتراح «خذ استراحة 2 دقيقة وتمرين تنفس» ثم يعود للاختبار. 

Plan

القواعد السلوكية للحركة داخل التطبيق (Business Rules)

لا تُظهر محتوى تقني مكثف عندما تكون مؤشرات التركيز منخفضة — تقدم بدائل مختصرة.

عند اكتشاف نمط فشل متكرر في نوع معين من المسائل، زوّد المستخدم بسلسلة تدريبات مُركّزة لهذا النمط.

تُصدر تقارير أداء دورية قابلة للتنزيل أو العرض داخل الحساب. 

Plan

حالات خطأ وحواف (Edge cases)

إجابة لا مكتملة في الاختبار → تُحفّظ الحالة وتعرض «أكمل لاحقاً».

فقدان الشبكة أثناء إجراء اختبار → تحفظ المحاولة مؤقتاً وتعرض إشعار واضح.

نتيجة نفسية عالية القلق → تعرض تحذير وتشجع على استراحة أو مسار تهدئة. 

Plan

متطلبات المحتوى (Content Matrix)

أسئلة نفسية مُعدة مسبقًا (مقاييس قصيرة قابلة للتعديل).

بنوك أسئلة تقنية مُصنفة حسب المهارة والمستوى.

فيديوهات قصيرة ونصوص تعليمية لكل وحدة.

نصوص تحفيزية وموجهات نفسية قصيرة لكل نتيجة/حدث. 

Plan

معايير القبول (Acceptance Criteria) — أمثلة قابلة للاختبار

بعد إكمال onboarding، يظهر للمستخدم بطاقة ملخص تحتوي على Mindprint وCoding Genome و3 توصيات قابلة للتنفيذ.

عند انتهاء درس + اختبار، تُعرض نتيجة واضحة مع شرح خطأ/صواب وخريطة ضعف محدثة.

لو أظهر الفحص الصباحي طاقة منخفضة، يقترح التطبيق مسار مختصر واحد على الأقل يمكن إكماله خلال 10 دقائق. 

Plan

قياس النجاح (KPIs)

نسبة إتمام المسارات الأولى خلال 30 يوم.

تحسّن متوسط نتائج الاختبارات لكل مستخدم شهريًا.

نسبة تراجع مستوى القلق لمستخدمي التركيز العالي.

معدل الاحتفاظ (retention) بعد 30 و90 يوم. 

Plan

مخرجات التسليم للـ Software Engineer (ما يُسلَّم إليه حرفياً)

وثيقة متطلبات وظيفية كاملة (FRD) — هذا الملف: تعريف الميزات، واجهات المستخدم المطلوبة، بيانات المخرجات لكل شاشة، حالات حواف، قواعد عمل. 

Plan

تدفقات المستخدم (User Flows) ورسم تخطيطي للشاشات (سردي ووظيفي).

مصفوفة محتوى: بنوك الأسئلة، النصوص، عناوين الدورات، توصيف التصنيفات.

Acceptance Criteria + Test Cases لكل ميزة أساسية.

ملف المحتوى (عربي/إنجليزي) للعناصر النصية الأساسية.

ملاحظة مهمة: لا يحتوي أي من المواد السابقة على توجيهات تقنية أو قائمة أدوات تنفيذية أو مخططات بنية تقنية — فقط فهم وظيفي وتجربة مستخدم مفصّلة كما طلبت. 

Plan

عناصر إضافية موصى بها للتسليم (غير تقنية)

نسخة PDF من وثيقة UX لعرض المنتج. 

Plan

مجموعة سيناريوهات اختبارية يدوية لكل رحلة مستخدم.

قائمة أولويات إطلاق (MVP): Onboarding، Dashboard، محرك الدورات، نظام التقييم، تقارير أساسية.
